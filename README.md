# 🔤 Генератор Текста по Регулярному Выражению

Инструмент для генерации слов из regex-паттернов. Помогает в тонкой настройке регулярных выражений — видите все возможные комбинации, которые ловит ваш паттерн, и корректируете его под задачу.

## ✨ Возможности

- 🔄 Генерация всех комбинаций по паттерну
- ✅ **Принятые слова** — подходят под паттерн
- ❌ **Отклонённые слова** — заблокированы негативным lookahead/lookbehind
- 🔒 Поддержка Unicode escape (`\u0020` → пробел)
- 📝 Сохранение паттернов в localStorage (сохраняются при обновлении страницы)
- 📂 Загрузка паттернов из файла (.js, .txt) — извлекаются только строки `const PATTERN = '...';`
- 📋 Вставка из буфера обмена
- 📄 Копирование паттернов в буфер обмена
- 🎯 Простой интерфейс с иконками и авторасширением текстового поля
- ⚡ Быстрая генерация через JSON API
- 🛑 Возможность отменить генерацию, если она зависла

## 📖 Использование

### 1. Введи паттерн

Введите паттерны в текстовое поле (каждая строка — отдельный паттерн). Нажми **Сгенерировать** для проверки.

Поле пустое по умолчанию. Можно загрузить паттерны из файла, вставить из буфера или ввести вручную.

**Примеры паттернов:**

```regex
# Простой паттерн
[пП][иИеЕ]?[зЗ3жЖ][дДтТ]

# С негативным lookahead (не после пробела/а/о)
[пП][иИ][жЖ](?![\u0020аАоО])

# С негативным lookbehind (не после у/э/ти)
(?<![уУэЭ]|[тТ][иИ])[пП][еЕиИ]?[зЗ3жЖ][дДтТ]

# С квантификаторами
[а-я]{3,5}
```

### 2. Загрузка паттернов из файла

Нажми **📂 Загрузить** и выбери файл (.js или .txt) с паттернами.

**Поддерживаемый формат:**
- Только строки вида `const PATTERN_NAME = '...';` (или `"..."`)
- Весь остальной JavaScript-код (функции, переменные, `new RegExp()` и т.д.) пропускается

**Пример файла:**
```javascript
const PATTERN_BLYAD = '(?<![а-яё]|...)[бБ6][лЛ][яЯ][тТ]';
const PATTERN_HUI = '(?<![уУ]|...)[хХ][уУ][йЙ][лЛ]';
// Остальной код будет пропущен при загрузке
const regex = new RegExp(...); // Эта строка не попадет в генерацию
```

### 3. Сгенерируй

Нажми **Сгенерировать**:
- **Верхнее поле (✅)** — принятые слова (подходят под паттерн)
- **Нижнее поле (❌)** — отклонённые слова (заблокированы lookahead/lookbehind)

Если генерация зависла, нажми **Отменить** (кнопка меняет текст во время генерации).

### 4. Работа с паттернами

- **📋 Вставить** — вставить паттерны из буфера обмена
- **📄 Копировать** — скопировать отредактированные паттерны в буфер
- **✕ Очистить** — полностью очистить поле ввода (удаляется и из localStorage)

### 5. Настройки фильтрации

- **Исключить заглавные** — убрать слова с заглавными буквами
- **Только Кириллица** — оставить только кириллические буквы
- **Исключить цифры** — убрать слова с цифрами
- **Исключить спецсимволы** — убрать слова со спецсимволами

## 🧪 Примеры

### Пример 1: Простой паттерн

**Паттерн:** `[пП][иИ]?[сС][тТ](?![оО])`

| Принятые | Отклонённые |
|----------|-------------|
| пст | псто |
| пист | писто |
| Пст | Псто |

### Пример 2: С lookahead

**Паттерн:** `[пП][иИ][жЖ](?![\u0020аАоО])`

| Принятые | Отклонённые |
|----------|-------------|
| пиж, пиЖ, Пиж | пиж , пижа, пижо |
| пИж, ПиЖ | пижк, пижн |

**Важно:** Слова с пробелом в конце (`пиж `) сохраняются для тестирования расширений.

### Пример 3: С lookbehind

**Паттерн:** `(?<![уУэЭ]|[тТ][иИ])[пП][еЕиИ]?[зЗ3жЖ][дДтТ]`

| Принятые | Отклонённые |
|----------|-------------|
| пизд, пезд, пиз | упезд, эпизд, типизд |
| пижд, пизт | — |

## 🔧 API

### GET /generate

Возвращает JSON с принятыми и отклонёнными словами.

**Параметры:**

| Параметр | Тип | Описание |
|----------|-----|----------|
| `patterns` | string[] | Массив regex паттернов (обязательно) |
| `exclude_uppercase` | bool | Исключить заглавные (default: true) |
| `exclude_latin` | bool | Исключить латиницу (default: true) |
| `exclude_digits` | bool | Исключить цифры (default: true) |
| `exclude_special` | bool | Исключить спецсимволы (default: true) |

**Ответ (один паттерн):**
```json
{
  "accepted": ["пиж", "пиЖ", "Пиж"],
  "rejected": ["пиж ", "пижа", "пижо", "пижк"]
}
```

**Ответ (множество паттернов):**
```json
{
  "results": [
    {
      "accepted": ["бля", "бляд"],
      "rejected": ["ублят", "роблят", ...],
      "error": null
    },
    {
      "accepted": ["бля", "бляд", "ьля", "ьляд"],
      "rejected": ["бляе", "блял", ...],
      "error": null
    }
  ]
}
```

**Пример запроса:**
```bash
curl "http://localhost:8080/generate?patterns=%28%3F%21%5B%D1%83%D0%A3%5D...%29"
```

## 🎯 Для чего это?

### Тонкая настройка паттернов

Увидьте все возможные комбинации, которые ловит ваш паттерн. Это помогает:
- ✅ Проверить, что паттерн ловит нужные слова
- ❌ Убедиться, что не ловятся лишние комбинации
- 🎯 Подобрать правильные lookahead/lookbehind ограничения

### Отладка регулярных выражений

Вместо проверки паттерна на парах тестовых примеров, вы видите полный список сгенерированных слов и можете скорректировать паттерн для нужного результата.

## 📝 Поддерживаемые конструкции

| Конструкция | Пример | Описание |
|-------------|--------|----------|
| Класс символов | `[аА@]` | Один из: а, А, @ |
| Диапазон | `[а-я]` | Любая буква от а до я |
| Опционально | `?` | 0 или 1 раз |
| Звёздочка | `*` | 0-5 раз |
| Плюс | `+` | 1-5 раз |
| Негативный lookahead | `(?!те)` | Не должно следовать "те" |
| Негативный lookbehind | `(?<!у)` | Не должно precede "у" |
| Unicode escape | `\u0020` | Пробел (U+0020) |

## 🐛 Известные ограничения

- Не поддерживает backreferences (`\1`, `\2`)
- Не поддерживает capture groups для извлечения
- Максимум 10 повторений в квантификаторах (для производительности)
- Не поддерживает рекурсивные паттерны
- `\s` не поддерживается (используйте `\u0020` для пробела)
