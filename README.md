# 🔤 Генератор Текста по Регулярному Выражению

Генератор слов по regex-паттернам с поддержкой негативных lookahead и lookbehind. Идеален для тестирования систем модерации и проверки регулярных выражений.

## ✨ Возможности

- 🔄 Генерация всех комбинаций по паттерну
- ✅ **Принятые слова** — подходят под паттерн
- ❌ **Отклонённые слова** — заблокированы негативным lookahead/lookbehind
- 🔒 Поддержка Unicode escape (`\u0020` → пробел)
- 📝 Сохранение пробелов в словах (для тестирования расширений)
- 🎯 Простой интерфейс без лишних настроек
- ⚡ Быстрая генерация через JSON API

## 📖 Использование

### 1. Введи паттерн

По умолчанию уже загружен тестовый паттерн. Нажми **Сгенерировать** для проверки.

**Примеры паттернов:**

```regex
# Простой паттерн
[пП][иИеЕ]?[зЗ3сС][дДтТ]

# С негативным lookahead (не после пробела/а/о)
[пП][иИ][жЖ](?![\u0020аАоО])

# С негативным lookbehind (не после у/э/ти)
(?<![уУэЭ]|[тТ][иИ])[пП][еЕиИ]?[зЗ3жЖ][дДтТ]

# С квантификаторами
[а-я]{3,5}
```

### 2. Сгенерируй

Нажми **Сгенерировать** или `Enter`:

- **Левое поле (✅)** — принятые слова (без запрещённых суффиксов/префиксов)
- **Правое поле (❌)** — отклонённые слова (с запрещёнными суффиксами/префиксами)

### 3. Проанализируй

| Слово | Где | Почему |
|-------|-----|--------|
| `пиж` | ✅ Принятые | После `ж` ничего нет |
| `пиж ` | ✅ Принятые | После `ж` пробел (защищено lookahead) |
| `пижа` | ✅ Принятые | После `ж` `а` (защищено lookahead) |
| `пижк` | ❌ Отклонённые | После `ж` другая буква |
| `упезд` | ❌ Отклонённые | Перед `п` есть `у` (lookbehind) |
| `типизд` | ❌ Отклонённые | Перед `п` есть `ти` (lookbehind) |

## 🧪 Примеры

### Пример 1: Простой паттерн

**Паттерн:** `[пП][иИ]?[сС][тТ](?![оО])`

| Принятые | Отклонённые |
|----------|-------------|
| пст | псто |
| пист | писто |
| Пст | Псто |

### Пример 2: С lookahead

**Паттерн:** `[пП][иИ][жЖ](?![\u0020аАоО])`

| Принятые | Отклонённые |
|----------|-------------|
| пиж, пиЖ, Пиж | пиж , пижа, пижо |
| пИж, ПИЖ | пижк, пижн |

**Важно:** Слова с пробелом в конце (`пиж `) сохраняются для тестирования расширений.

### Пример 3: С lookbehind

**Паттерн:** `(?<![уУэЭ]\|[тТ][иИ])[пП][еЕиИ]?[зЗ3жЖ][дДтТ]`

| Принятые | Отклонённые |
|----------|-------------|
| пизд, пезд, пиз | упезд, эпизд, типизд |
| пижд, пизт | — |

## 🔧 API

### GET /generate

Возвращает JSON с принятыми и отклонёнными словами.

**Параметры:**

| Параметр | Тип | Описание |
|----------|-----|----------|
| `pattern` | string | regex паттерн (обязательно) |

**Ответ:**

```json
{
  "accepted": ["пиж", "пиЖ", "Пиж"],
  "rejected": ["пиж ", "пижа", "пижо", "пижк"]
}
```

**Пример запроса:**

```bash
curl "http://localhost:8080/generate?pattern=%5BпП%5D%5BиИ%5D%5BжЖ%5D%28%3F%21%5B%5Cu0020аАоО%5D%29"
```

## 🎯 Для чего это?

### Тестирование модерации

Проверь что твой regex-фильтр:

- ✅ Ловит все запрещённые слова
- ❌ Не блокирует легитимные (пижама, команда, мандарин)
- 🎯 Не пропускает обходные варианты

### Отладка паттернов

Увидь все возможные комбинации которые ловит твой паттерн, а не только тестовые примеры.

### Обучение

Пойми как работают:

- Классы символов `[аА@]`
- Негативные lookahead `(?!...)`
- Негативные lookbehind `(?<!...)`
- Опциональные элементы `?`
- Unicode escape `\u0020`

## 📝 Поддерживаемые конструкции

| Конструкция | Пример | Описание |
|-------------|--------|----------|
| Класс символов | `[аА@]` | Один из: а, А, @ |
| Диапазон | `[а-я]` | Любая буква от а до я |
| Опционально | `?` | 0 или 1 раз |
| Звёздочка | `*` | 0-5 раз |
| Плюс | `+` | 1-5 раз |
| Негативный lookahead | `(?!те)` | Не должно следовать "те" |
| Негативный lookbehind | `(?<!у)` | Не должно precede "у" |
| Unicode escape | `\u0020` | Пробел (U+0020) |

## 🐛 Известные ограничения

- Не поддерживает backreferences (`\1`, `\2`)
- Не поддерживает capture groups для извлечения
- Максимум 10 повторений в квантификаторах (для производительности)
- Не поддерживает рекурсивные паттерны
- `\s` не поддерживается (используйте `\u0020` для пробела)
