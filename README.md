# 🔤 Генератор Текста по Регулярному Выражению

Генератор слов по regex-паттернам с поддержкой негативных lookahead и фильтрации. Идеален для тестирования систем модерации и проверки регулярных выражений.

## ✨ Возможности

- 🔄 **Генерация всех комбинаций** по паттерну
- ✅ **Принятые слова** — подходят под паттерн
- ❌ **Отклонённые слова** — заблокированы негативным lookahead
- 🎛 **Гибкая фильтрация**:
  - Исключить заглавные буквы
  - Исключить латиницу
  - Исключить цифры
  - Исключить спецсимволы
- 🔒 **Поддержка Unicode** (`\u00D7` → `×`)
- ⏸ **Остановка генерации** в любой момент
- 📊 **Прогресс-бар** в реальном времени
- 💾 **Сохранение настроек** в localStorage

## 🚀 Установка

### Требования
- Go 1.21+
- OpenSSL (для HTTPS)

### Быстрый старт

```bash

- **HTTP**: http://localhost:8080
- **HTTPS**: https://localhost:8443 (прими самоподписанный сертификат)

## 📖 Использование

### 1. Введи паттерн

**Примеры:**

```regex
# Простой паттерн
[пП][иИеЕ]?[зЗ3сС][дДтТ]

# С негативным lookahead
[хХxX\u00D7][уУyY][лЛйЙёЁяЯеЕeEаАaA@][иИтТ]?(?![еЕтТгГсС]|[иИ][тТ][еЕсС])

# С квантификаторами
[а-я]{3,5}
```

### 2. Настрой фильтры

- ☑️ **Исключить заглавные** — только строчные буквы
- ☑️ **Исключить латиницу** — только кириллица
- ☑️ **Исключить цифры** — только буквы
- ☑️ **Исключить спецсимволы** — только алфавит
- ☑️ **Отключить Unicode** — не декодировать `\uXXXX`
- ☑️ **Генерировать негативные** — показать отклонённые слова

### 3. Сгенерируй

Нажми **Сгенерировать** и наблюдай за процессом:

- **Левое поле** — принятые слова (без запрещённых суффиксов)
- **Правое поле** — отклонённые слова (с запрещёнными суффиксами)

### 4. Проанализируй

Если в правом поле много слов — твой паттерн ловит лишнее! Уточни regex.

## 🔧 API

### GET /generate

Server-Sent Events (SSE) поток для генерации слов.

**Параметры:**
- `pattern` (string) — regex паттерн
- `exclude_uppercase` (bool) — исключить заглавные
- `exclude_latin` (bool) — исключить латиницу
- `exclude_digits` (bool) — исключить цифры
- `exclude_special` (bool) — исключить спецсимволы
- `disable_unicode` (bool) — отключить Unicode
- `generate_negative` (bool) — генерировать отклонённые
- `session_id` (string) — ID сессии для отмены

**События:**
```json
{ "type": "word", "word": "хули" }
{ "type": "rejected", "word": "хулите" }
{ "type": "progress", "progress": 50, "count": 100, "rejected_count": 50, "total": 300 }
{ "type": "complete", "count": 150, "rejected_count": 150, "total": 300 }
{ "type": "error", "error": "незакрытая скобка [" }
```

### POST /cancel

Отмена генерации.

**Тело запроса:**
```json
{ "session_id": "1234567890" }
```

## 🎯 Для чего это?

### Тестирование модерации

Проверь что твой regex-фильтр:
- ✅ Ловит все запрещённые слова
- ❌ Не блокирует легитимные (команда, мандарин)
- 🎯 Не пропускает обходные варианты

### Отладка паттернов

Увидь **все возможные комбинации** которые ловит твой паттерн, а не только тестовые примеры.

### Обучение

Пойми как работают:
- Классы символов `[аА@]`
- Квантификаторы `{1,2}`
- Негативные lookahead `(?!...)`
- Опциональные элементы `?`

## 📝 Поддерживаемые конструкции

| Конструкция | Пример | Описание |
|------------|--------|----------|
| Класс символов | `[аА@]` | Один из: а, А, @ |
| Диапазон | `[а-я]` | Любая буква от а до я |
| Квантификатор | `{2,4}` | От 2 до 4 повторений |
| Опционально | `?` | 0 или 1 раз |
| Звёздочка | `*` | 0-5 раз |
| Плюс | `+` | 1-5 раз |
| Негативный lookahead | `(?!те)` | Не должно следовать "те" |
| Unicode escape | `\u00D7` | Символ × (умножение) |

## 🐛 Известные ограничения

- Не поддерживает backreferences (`\1`, `\2`)
- Не поддерживает capture groups для извлечения
- Максимум 10 повторений в квантификаторах (для производительности)
- Не поддерживает рекурсивные паттерны
